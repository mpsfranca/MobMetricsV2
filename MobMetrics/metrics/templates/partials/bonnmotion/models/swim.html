How to use SWIM
Node radius (-r) Required

Radius used for node contact detection and for deriving the SWIM cell size.

  • Units normalized distance units / communication radius
  • Enter Positive number
Cell distance weight (-c) Required

Weight that balances preference for nearby cells against popularity of cells already seen.

  • Enter Usually between 0 and 1
    Example: 0.8 — higher values favor nearby destinations more strongly.
Node speed multiplier (-m) Required

Multiplies movement distance to obtain node speed.

  • Units inverse time scale / speed factor
  • Enter Positive number
    Example: 1.0
Waiting time exponent (-e) Required

Exponent of the upper-bounded power-law distribution used for waiting times.

  • Enter Number greater than 1
    Example: 1.45
    Smaller exponents create heavier tails; larger ones reduce long waits more quickly.
Waiting time upper bound (-u) Required

Maximum allowed waiting time. After drawing from the power-law distribution, the code truncates the result so that it never exceeds this bound.

  • Units time units / seconds
  • Enter Positive number
    Example: 3600
Example:
-r 0.05 -c 0.8 -m 1.0 -e 1.45 -u 3600
— where -r, -c, -m, -e, and -u are the SWIM flags.

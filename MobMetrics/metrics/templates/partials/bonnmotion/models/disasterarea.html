How to use DisasterArea
Average no. of nodes per group (-a) Optional

Average number of nodes per group. In this model, this parameter is kept for compatibility, but group sizes actually depend on the catastrophe areas.

  • Units number of nodes
  • Enter Positive number
    Default: 3.0
Catastrophe area (-b) Required

Defines one catastrophe area. This parameter can be used multiple times to create several catastrophe areas, and the model depends on these areas to build the scenario.

  • Enter Area definition
    Repeat this parameter for multiple catastrophe areas.
Group change probability (-c) Optional

Probability of changing groups. In this model, this parameter is not effectively considered, because groups belong to areas.

  • Units probability
  • Enter Number from 0 to 1
    Default: 0.0
Max catastrophe areas (-e) Optional

Maximum number of catastrophe areas allowed in the scenario.

  • Units integer
  • Enter Integer ≥ 1
    Default: 4
Max. distance to group center (-r) Optional

Maximum distance a node may move away from its group reference point.

  • Units meters (m)
  • Enter Positive number
    Default: 2.5
Group size standard deviation (-s) Optional

Variation around the average group size. In this model, this parameter is kept for compatibility, but group sizes actually depend on the catastrophe areas.

  • Units number of nodes
  • Enter Non-negative number
    Default: 0
Obstacle for only one group (-O) Optional

Defines an obstacle that applies only to one specific group, identified by the last value in the parameter list.

  • Enter Obstacle definition + group id
Write vis. info to file & show movements (-w) Optional flag

Writes movement visualization information to file and enables movement output.

  • This is a flag and does not require a numeric value.
Write vis. info to file & show vis.graph (-v) Optional flag

Writes visibility-graph information to file and enables visibility-graph output.

  • This is a flag and does not require a numeric value.
Do not knock over pedestrians (-K) Optional flag

Prevents the model from treating pedestrians as knock-over candidates; in the code, this activates the no-knock-over behavior for areas beside APP.

  • This is a flag and does not require a numeric value.
Example:
-b 10,10,40,10,40,40,10,40,100,100,120,120,0,2,1 -e 4 -r 2.5 -O 25,150,100,150,100,200,25,200,3 -w -v -K
